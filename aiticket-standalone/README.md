# aiticket-standalone · 本地 Jira 智能工单助手

> 在你自己的电脑上，跑一个"智能工单看板"：看 Jira 工单、AI 帮你写回复、用你自己的知识库找答案、越用越准。
> **你不用懂技术，也不用记命令——在 OpenClaw / WorkBuddy 里用大白话跟它说就行。**

---

## 它能帮你做什么

装好后浏览器打开一个网页（在你自己电脑上），你会看到：

- **🗂 智能看板**：你负责的 Jira 工单一目了然
- **🤖 智能回复**：点一个工单，AI 结合"知识库 + 历史相似工单"给你处理建议，改改就能用
- **📚 知识库问答**：把你的文档（手册 / FAQ / 方案）放一个文件夹，就能语义搜索
- **🧠 越用越准**：你采纳/修改回复后它会记住

全部跑在**你自己电脑上**，数据不外传。

---

## 怎么用？—— 你只管说，它来做

你不用敲命令。在 OpenClaw / WorkBuddy 里，像聊天一样说下面这些话就行（每句下面是它会替你做的事，**全程有进度提示**）：

### 1. 安装

> 💬 你说：「**帮我把 aiticket 装上**」 或「**安装本地工单助手**」

🤖 它会自动：
1. 检查 **Python、git**——缺了就**自动安装**（显示安装进度）
2. 建好运行环境、装好程序（显示进度）
3. **下载中文 AI 模型**（约 120MB，仅一次，显示下载进度）
4. 启动服务，给你一个网址（一般是 `http://127.0.0.1:18080`）

> 🟢 装完它会告诉你："装好了，浏览器打开 http://127.0.0.1:18080 就能看到看板"。**不用登录、不用填地址**——都默认配好了。

### 2. 让它看到你的 Jira 工单

> 💬 你说：「**帮我连上 Jira**」

🤖 它会引导你装一个浏览器小插件（一次性），自动把你浏览器里已登录的 Jira 会话同步过来——之后看板就能显示真实工单了。（也支持手动粘贴，它会一步步教你。）

### 3. 设你负责的项目 → 自动导入近一年历史

> 💬 你说：「**我负责 LCZX 项目**」 或「**把默认项目设成 BIP**」

🤖 它会记下你的项目，并在你连好 Jira 后**自动把这个项目最近 12 个月的历史工单拉下来建索引**（这样 AI 写回复能参考相似的老工单）。
> 📊 这步在后台跑，**有进度百分比**；你也可以随时说「**看下历史导入进度**」。

### 4. 选你的知识库文件夹 → 自动解析

> 💬 你说：「**把我桌面的 docs 文件夹设成知识库**」

🤖 它会把那个文件夹设为知识库，并**立刻自动解析**（扫描 → 切片 → 向量化，支持 `.md/.docx/.pdf/.xlsx/.pptx/.txt` 等）。
> 📊 解析**有进度提示**（已处理几个文件、生成多少片段）；跑完就能在"知识库"页搜索、AI 回复也会引用它。

### 5. 日常

> 💬 「**工单看板怎么样了 / 服务还在跑吗**」→ 查看运行状态
> 💬 「**重启一下工单助手**」→ 重启服务
> 💬 「**给我看下日志**」→ 显示运行日志
> 💬 「**更新到最新版**」→ 拉新版并重启
> 💬 「**重新导入一下 LCZX 的历史工单**」→ 手动重跑历史导入（带进度）
> 💬 「**卸载工单助手**」→ 停服务并清理

> 这些大白话，OpenClaw / WorkBuddy 都能听懂并自动执行（对应的内部命令见同目录 `SKILL.md`）。

## 装它之前，电脑要准备什么？

**几乎不用准备。** 联网即可——Python、git 缺了它会自动装。
（如果你愿意手动装也行：Python 3.11+ 与 git；但通常交给它自动处理更省事。）
> 第一次安装需要联网下载依赖和一个 AI 模型；之后可离线使用。

---

## 所有"下载 / 安装 / 解析"都看得见进度

这是专门为"非技术用户"设计的——任何会等待的操作都有提示，你不会面对黑屏干等：

| 操作 | 进度表现 |
|------|---------|
| 装 Python / git / 环境 | 分步进度 `[1/4]…[4/4]` + ✓/⚠ |
| 装程序依赖 | 逐个包的安装进度 |
| 下载 AI 模型（约 120MB） | 下载百分比进度条 |
| 导入历史工单 | 百分比（已处理 N / 共 M 条） |
| 解析知识库 | 步骤 + 已处理文件数 / 生成片段数 |

## 进阶：接进 Claude Code / OpenClaw（可选）

让 AI 助手把本服务当"工具"，用它自己的大模型来生成回复（你这边不用配大模型）。
跟它说「**帮我把 aiticket 接进 MCP**」，它会打印好配置片段；或手动：

```bash
uv pip install -r tools/requirements-mcp.txt
```
```json
{
  "mcpServers": {
    "aiticket": {
      "command": "~/.aiticket/venv/bin/python",
      "args": ["<本 skill 路径>/tools/mcp_server.py"],
      "env": { "AITICKET_HOME": "~/.aiticket" }
    }
  }
}
```
（Windows 用 `~\.aiticket\venv\Scripts\python.exe`。）

## 它装到哪？占多大？

- skill 本体很小（约 9MB）。装到电脑后会多出：运行环境约 250MB + AI 模型约 120–460MB（只下一次，多项目共用）。这是"本地 AI 检索"的必要成本。
- **想更省空间**：跟它说用"云端向量"或"离线模式"（环境变量 `EMBEDDING_PROVIDER=api` 或 `hash`），就不下本地模型。
- 你的**管理员/会话数据**在 `~/.aiticket/data`，更新/重装都不丢；向量/缓存可随时重建。

## 遇到问题？直接问它

> 💬 「**看板里没工单**」→ 多半是 Jira 会话过期，它会让你重新连一下
> 💬 「**装到一半卡住了**」→ 它会看日志找原因（端口被占就换端口）
> 💬 「**下载模型太慢**」→ 可让它切"离线模式"先用起来

手动排查：`python tools/aiticket_ctl.py logs --err`（看错误日志）、`python tools/aiticket_ctl.py status`（查健康）。

## 手动安装（给愿意敲命令的人）

```bash
# 一条命令：自动装 Python/git/环境 + 装服务（全程进度）
bash tools/bootstrap_env.sh          # macOS / Linux
# Windows：
powershell -ExecutionPolicy Bypass -File tools\bootstrap_env.ps1
```
若环境已具备，也可直接 `python tools/install.py`。

## 卸载

```bash
python tools/aiticket_ctl.py stop    # 先停服务（会一并注销开机自启）
rm -rf ~/.aiticket                    # 删安装目录（含数据；不想删数据可跳过）
```
（Windows 删除 `%USERPROFILE%\.aiticket`。）
