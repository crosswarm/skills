## Design Context

### Users
产品经理、需求分析师。使用场景：长时间专注撰写需求文档，需要平静高效的写作环境。
核心任务：起草 → AI 生成 → 精化 → 评审需求文档。

### Brand Personality
Professional · Efficient · Clear（专业 · 高效 · 清晰）

### Aesthetic Direction
活跃清新（Active & Fresh）
- 参考：Notion / Craft / Bear
- 白色背景，Amber #f59e0b 作为模块强调色（比系统基础色 #d97706 更亮）
- 清晰可见的分割线，充足的白空间
- 激活态：Notion 式左侧 4px Amber 竖线 + 浅 Amber 背景
- 暗色模式：跟随全局 data-theme 开关

### Design Principles
1. Content First — 最大化写作/阅读区域，减少 Chrome 干扰
2. Active & Fresh — Amber 作点缀，大量白空间，Notion 式简洁感
3. Immediate Feedback — 保存状态、版本标记始终清晰可见
4. Familiar Patterns — 工具栏图标符合业界标准（GitHub/Notion 风格）
5. System Consistent — 沿用现有设计 token，尊重暗色模式

### Editor
StackEdit.js (stackedit-js@1.0.7)
- 预览：openFile(file, true) 静默渲染，返回 file.content.html
- 编辑：openFile(file) 打开全屏 overlay 编辑器（需访问 stackedit.io 外网）
- 降级：StackEdit 超时 (8s) 时自动 fallback 到 marked.js
