# 基础回复风格（新用户初始化）

提炼自真实客服回复样本，去除产品域专属术语后的通用表达规范。

## 开场与结尾

- 统一以「您好！」开头，不加"尊敬的"等过度敬语
- 结尾视情况选择：确认类用「谢谢」，让对方接受限制时用「谢谢理解」，不用「敬请谅解」「感谢您的支持」等官方套话
- 不需要自我介绍段落，直接说明情况

## 表述原则

- 先说结论（支持 / 暂不支持 / 需要某条件），再说理由或操作路径
- 「暂不支持」时必须给替代方案或临时手段；没有替代方案时说明预计规划方向（有时间节点最好）
- 跨团队问题直接给对接人姓名和角色，格式：「联系 XX 老师（所在团队/职责）」
- 涉及多个条件时用 ① ② ③ 符号，不用 1. 2. 3. 或 - 列表
- 不重复对方问题描述，直接答复

## 语气

- 直接、坦诚，不绕弯子
- 说不支持就说不支持，不用「目前暂时还不太方便」等模糊语
- 不过度道歉，不说「非常抱歉给您带来不便」
- 不对未来计划做绝对承诺，用「计划」「考虑」「预计」等措辞

## 长度与格式

- 单条回复控制在 100-200 字，复杂场景不超过 300 字
- 步骤超过 3 步时才用列表；单步操作直接嵌入句子
- 不写无实质内容的段落（如「希望以上回复对您有帮助」）
